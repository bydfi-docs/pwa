# PWA Screenshots 截图说明

## 📸 截图要求

为了在 Android Chrome 安装时显示应用预览，需要准备以下截图：

### 文件列表
- `home.png` - 首页（市场行情）
- `market.png` - 行情页面
- `trade.png` - 交易页面
- `assets.png` - 资产页面

### 截图规格
- **尺寸**：540x960 px（或更高分辨率，保持 9:16 比例）
- **格式**：PNG（推荐）或 JPEG
- **设备**：在真实手机或浏览器开发者工具的移动模式下截图

## 🎯 如何制作截图

### 方法1：使用浏览器开发者工具（推荐）

1. 在 Chrome 中打开应用：`npm run dev`
2. 按 F12 打开开发者工具
3. 点击设备切换按钮（📱图标）或按 Ctrl+Shift+M (Windows) / Cmd+Shift+M (Mac)
4. 选择设备：iPhone 12 Pro 或类似尺寸（390x844）
5. 导航到每个页面并截图

### 方法2：使用在线工具

- 访问应用的各个页面
- 使用截图工具（如 Chrome 扩展 "Full Page Screen Capture"）
- 调整尺寸至 540x960

### 方法3：真实设备截图

1. 在手机上打开应用
2. 截取每个页面的屏幕截图
3. 使用图片编辑工具调整至推荐尺寸

## 🎨 截图建议

- **突出核心功能**：展示最吸引用户的界面
- **使用真实数据**：避免空白页面
- **保持一致性**：统一的风格和布局
- **高质量**：清晰、无模糊
- **顺序**：按用户使用流程排列

## 📱 展示效果

Android Chrome 会在安装提示对话框中以轮播图的形式展示这些截图，用户可以左右滑动查看。

## 🔧 如何测试

1. 准备好截图后，放入 `public/screenshots/` 目录
2. 运行 `npm run build`
3. 部署到服务器或使用 `npm run preview`
4. 在 Android Chrome 中访问，触发安装提示
5. 查看截图展示效果

## 📝 注意事项

- 截图文件大小建议控制在 500KB 以内
- 至少提供 1 张，最多 8 张截图
- 截图会被压缩，建议使用高质量原图
- iOS Safari 目前不支持此功能
